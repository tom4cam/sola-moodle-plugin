# SOLA v3.9.6 — Release Notes, Feature Summary, and 20-minute Admin Walkthrough

## Part 1: Release Notes (21 April 2026)

v3.9.6 extends the RAG indexer to cover video transcripts, uploaded files (PDF and DOCX), H5P interactives, SCORM packages, and embedded third-party interactives (Synthesia, YouTube, Articulate, Genially). It also bundles several deploy and hook fixes from the prior week and introduces a standardised release-notes helper script.

### New RAG content extractors

- **Uploaded files (`mod_resource`).** PDF via `pdftotext` (shell-out to `poppler-utils`) with auto-detection of the binary path. DOCX via native PHP `ZipArchive`, no Composer dependency. TXT and Markdown read directly.
- **H5P (`mod_h5p`).** Walks H5P `content.json` recursively and indexes text-bearing fields (questions, answers, feedback, descriptions, hints, solutions, alt text, prompts).
- **SCORM (`mod_scorm`).** Unzips the package, indexes `imsmanifest.xml` titles/descriptions, every `.html` inside the package, and Articulate Storyline content strings from `story_content/*.js`. Off by default because SCORM parses can be expensive; size-capped at 100 MB per package (configurable via `rag_scorm_max_mb`).
- **Embedded iframes + paired transcripts.** Page and Book HTML is scanned for iframe embeds matching an admin-configurable host list (defaults: Synthesia, YouTube, `youtube-nocookie.com`, Articulate Review, Articulate User Content, Articulate 360, Genially). For each matched iframe SOLA searches both above and below, by DOM proximity, for the nearest transcript anchor matching an admin-configurable URL regex (default: `resources.saylor.org/transcripts/…`) and pairs them. When `rag_fetch_transcripts` is enabled, SOLA fetches each transcript with a real browser User-Agent and appends it under a section marker `[Transcript of embedded video/interactive: …]`. Cloudflare challenge pages are detected and skipped with a log notice. When the fetch is disabled, SOLA still writes a minimal `[Embedded video/interactive: …]` marker so the LLM knows the embed exists.

### New admin settings (grouped on the Settings page and mirrored on the RAG Admin page)

- `rag_extract_pdf` + `rag_pdftotext_path`
- `rag_extract_docx`
- `rag_extract_h5p`
- `rag_extract_scorm` + `rag_scorm_max_mb`
- `rag_fetch_transcripts`
- `rag_iframe_host_patterns` (one regex per line; `#` comments supported)
- `rag_transcript_url_pattern`

### RAG Admin page: Content sources status card

New table at the top of the RAG Admin page shows each extractor with a green Ready or grey Off badge plus a one-line reason (binary missing, disabled in settings, etc.) so admins can see at a glance why content isn't indexing.

### Deploy and hook fixes (bundled from the v3.9.5 hotfix window)

- `deploy_dev.py` tar excludes macOS AppleDouble sidecars (`._*`) so Dropbox-synced plugin source no longer poisons the Moodle AMD aggregate with binary garbage.
- `classes/hook_callbacks.php` uses `__DIR__` instead of `$CFG->dirroot` for the custom-avatar helper include. Fixes a v3.9.5 regression where the widget silently failed to inject on course pages for admins whose page did not globalise `$CFG` in the hook scope.

### Release helper

- `scripts/new_release_notes.py` seeds a templated `.drafts/v<N>-release-notes-and-walkthrough.md` with TODOs. Step one of every future release. See the new Release Process section of `CLAUDE.md`.

### Testing

- Jailbreak suite: 25 of 25 PASS.
- PHP lint clean on every modified file (5 new, 4 modified).
- Deployed and verified on dev.sylr.org, dev405.sylr.org, dev500.sylr.org (Moodle 5.0), and dev501.sylr.org (Moodle 5.1 with the `public/` layout).

Plugin version `2026042101`, release `3.9.6`.

## Part 2: Key SOLA Features

### Chat and AI backbone

SOLA is a Moodle local plugin embedding a floating AI learning coach on course pages. 11 chat providers supported: Claude, OpenAI, DeepSeek, Gemini, Ollama (local), MiniMax, Mistral, OpenRouter, xAI Grok, Moodle AI (`core_ai`), Custom OpenAI-compatible endpoints. Responses stream via SSE. Default AI Provider configured site-wide, overridable per course.

### Comparison providers (LLM picker for admins)

Multi-row admin table stores additional chat providers (provider_id, api_key, model list, per-row temperature) so admins can A/B across providers from inside the widget. Admin only; students never see it.

### Voice

Multi-row voice registry (v3.9.5) stores one or more voice APIs (OpenAI and xAI Grok) with per-row API key, label, and default voices. Three active-provider selects pick which label drives Realtime, TTS, and STT independently. Voices: OpenAI (Shimmer, Alloy, Echo, Fable, Onyx, Nova) and Grok (Eve, Ara, Leo, Rex, Sal).

### RAG (v3.9.6 expanded)

Semantic search over course content with embedding-based retrieval. Now covers: `mod_page`, `mod_book`, `mod_assign`, `mod_forum`, `mod_label`, `mod_glossary`, `mod_quiz`, `mod_resource` (PDF/DOCX/TXT/MD), `mod_h5p`, `mod_scorm`. Iframe embeds paired with transcript links for Synthesia, YouTube, Articulate, and Genially. Disabled at the site level by default. RAG Admin page shows per-course index status and a content-source status card.

### Practice quizzes, study plans, reminders

Full quiz flow (AI-guided or topic-picker), interactive question cards, score summary. Study plans schedule email reminders; WhatsApp optional. Inactivity reminders after N days.

### Student learning profiles

After every N student messages, SOLA generates or refreshes a learning profile capturing strengths, weaknesses, style, and interests. Profile is injected into the system prompt.

### Analytics

Cross-course usage, provider comparison, student feedback, AI Analysis Chat with anonymized transcripts, scheduled email reports (daily/weekly/monthly), Token Cost and Analytics spend tracking.

### Testing Environment

Admin page creates a hidden testing course (`SOLATEST`, visible=0) and seeds fake students with synthetic AI conversations so admins can preview the Analytics Dashboard without touching real data.

### Safety and moderation

Off-topic detection, academic integrity nudges, wellbeing checks, prompt injection hardening, output safety filter that scrubs leaked prompts and API keys. Automated jailbreak suite: 25 prompts, 9 attack categories, gates on 25/25 PASS.

### i18n

46 language packs (en + 45 translations). STT and TTS in 46 languages. Resizable drawer (320 × 400 minimum), draggable, 2-state expand.

### Branding

Institution name, assistant display/short name, colour picker, procedurally generated SVG avatars, 10 built-in image avatars, admin-uploaded custom avatars (v3.9.5), per-user avatar preference.

### Delivery

CDN URL/version configurable, self-update admin page pulls from GitHub releases, Redash export for deeper BI.

### Surveys, rubrics, usability testing

In-chat surveys, practice scoring rubrics, task-based usability testing.

## Part 3: 20-minute Admin Walkthrough Script (conversational)

Total target time 20 minutes.

### 00:00 to 02:00 — Welcome and framing (2 min)

> "Welcome to SOLA. Today we'll configure the AI tutor, hook up voice, set up RAG so the tutor grounds its answers in your course content (including videos and PDFs now), preview analytics, and create a safe testing sandbox, all in about twenty minutes."

Navigate to **Site administration > Plugins > Local plugins > AI Course Assistant**. Note the Settings page on the left and the external admin pages (Testing Environment, RAG Admin, Analytics Dashboard, Token Cost, Survey Editor, Rubric Editor, Integrity) below.

### 02:00 to 05:00 — Default chat provider and API key (3 min)

> "First thing every admin does: pick a Default AI Provider and paste its API key."

Open Settings. Scroll to **AI Provider and Models**. Choose **Default AI Provider**. Click the password field labelled **API Key**, paste a key, Save. Mention Model Name, API Base URL, and Temperature briefly.

### 05:00 to 07:00 — Comparison providers (2 min)

Show the **Comparison providers** table. Add one row, pick a provider, paste a key, list models, set a per-row temperature. Save. Reload a course page, open the widget, demonstrate the LLM picker in the composer.

### 07:00 to 10:00 — Voice registry and Grok (3 min)

Go to **Voice providers**. Add a row with OpenAI labelled `openai-prod`, and a second with xAI labelled `grok-prod` with `eve` in both voice columns. Set **Active Realtime / TTS / STT** to the labels you want each capability to use. Save. Reload a course page with voice mode enabled and demonstrate that the Voice tab now talks to Grok for Realtime and TTS, Whisper for transcription.

### 10:00 to 13:00 — RAG content sources (new in v3.9.6) (3 min)

> "This is new in 3.9.6. RAG now reads from most of your course content, not just pages and books."

Scroll to **RAG content sources**. Walk through each checkbox: PDF, DOCX, H5P (all on), SCORM (off by default, we'll turn it on), and **Fetch transcripts for embedded videos/interactives**. Note the textarea of iframe host patterns and the transcript URL regex.

Open the **RAG Admin** page. Point out the **Content sources** status card, each extractor shows Ready or Off with a reason. If `pdftotext` is missing, the row will say so.

Click **Reindex course** on BUS101 and show the output: indexed, skipped, errors.

### 13:00 to 15:00 — Custom avatars and branding (2 min)

In **Branding and UI**, set Institution Name, Assistant Short Name, avatar colour. Upload a 256 × 256 PNG into **Custom avatars**. Save. Pick the custom upload as the default in the Avatar dropdown.

### 15:00 to 17:00 — Testing Environment + Analytics (2 min)

Go to **Testing Environment**. Click **Create hidden testing course**. Then **Seed students and chats** with defaults (15 users, 4 weeks). Wait for the success notification. Click **View Analytics for this course** and walk through the tabs: usage by provider, feedback, AI Analysis Chat.

### 17:00 to 18:30 — Safety and jailbreak test (1.5 min)

Mention prompt injection hardening and the output safety filter. Run `php admin/cli/jailbreak_test.php --courseid=2` if you have terminal access (returns 25 of 25 PASS).

### 18:30 to 19:30 — Drawer ergonomics and clear screen (1 min)

Open the widget. Drag the top-left corner to shrink to the 320 × 400 minimum. Click the eraser. Read the confirmation: "Clear the visible messages? Your full chat history stays saved." Confirm and show that analytics still has the full transcript.

### 19:30 to 20:00 — Next steps (0.5 min)

Recommend three things before going live: enable RAG + reindex one course (now including its videos and PDFs), configure a weekly AI Analysis Scheduled Report, and add a Comparison Provider to A/B your primary over a week.

Close: "If anything breaks, check Site admin > Notifications and GitHub issues at `saylordotorg/moodle-local_ai_course_assistant`."
