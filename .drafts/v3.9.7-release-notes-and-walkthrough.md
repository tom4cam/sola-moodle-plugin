# SOLA v3.9.7 — Release Notes, Feature Summary, and 20-minute Admin Walkthrough

## Part 1: Release Notes (21 April 2026)

v3.9.7 is a small transcript-fetcher hardening release. The RAG indexer now sends a full Chrome-desktop request header set (Accept, Accept-Language, Accept-Encoding, Sec-Fetch-\*, Sec-Ch-Ua-\*, Upgrade-Insecure-Requests, DNT, Priority) so naive Cloudflare Bot Fight rules that key off missing or out-of-order browser headers do not trip on us. It is defence in depth: the real bypass for the transcript host on `resources.saylor.org` remains the IP allowlist request pending with David Ta; this change reduces the odds of false-positive blocks on other transcript hosts and when the allowlist lands.

- `classes/content_extractor.php`: new `TRANSCRIPT_HEADERS` constant with a realistic Chrome/macOS header set; UA string bumped to Chrome 124 and stripped of the `SOLA-Indexer/1.0` token. `CURLOPT_ENCODING => ''` so curl can auto-decode gzip/br responses served to a real browser Accept-Encoding.
- No new settings, no lang strings changed, no DB changes.

### Why this is not a full bypass

Cloudflare's Bot Management also inspects TLS fingerprint (JA3), challenge cookies, and pixel-perfect header ordering that plain PHP curl cannot mimic. If a transcript host uses Managed Challenge or Super Bot Fight Mode, we will still see the "Just a moment..." interstitial and will skip cleanly with a debug notice. For those hosts the options are still the IP allowlist (preferred) or `curl-impersonate` / headless Chrome (heavy).

### Testing

- Jailbreak suite: 25 of 25 PASS.
- PHP lint clean on the one modified file.
- Deployed and verified on dev.sylr.org, dev405.sylr.org, dev500.sylr.org, dev501.sylr.org.
