# SOLA v3.9.8 — Release Notes, Feature Summary, and 20-minute Admin Walkthrough

## Part 1: Release Notes (21 April 2026)

v3.9.8 is a white-label + cost-visibility + Learning Radar upgrade. It separates author attribution from runtime branding so any institution can deploy and brand the plugin without touching code; breaks token spend out by workload category (Chat, Voice, RAG, Analytics); renames the admin chat tool to Learning Radar with at-a-glance metric chips and SOLA_NEXT follow-up suggestions.

### White-label cleanup

- New `CREDITS.md` at repo root crediting Tom Caswell and David Ta at Saylor University as original authors, with GPLv3 + open-source framing.
- New `.wiki/Saylor-Deployment-Notes.md` aggregating Saylor-specific infra notes (dev.sylr.org, Catalyst hosting, BUS101 smoke, Cloudflare allowlist); other wiki pages degreened for generic institutions.
- `@copyright` docblock across 81 PHP files updated to "2025-2026 Tom Caswell & David Ta / Saylor University". Attribution in code and git history; never surfaces in UI.
- Runtime defaults in `settings.php` switched from Saylor-specific to generic placeholders: `institution_name` → "Your Institution", `institution_short_name` → "Your Institution", `display_name` → "Online Learning Assistant", `short_name` → "Assistant", `rag_transcript_url_pattern` default cleared. Existing deployments are unaffected (DB-saved values win); only fresh installs see the new generics.
- `lang/en` audit: 37 Saylor/SOLA hits → 8 (remaining are defensible defaults like the system prompt template and UI placeholder copy).
- `README.md` gains a Credits section linking to CREDITS.md.

### Cost visibility: break out Voice / RAG / Analytics

- Token Cost page gains a **Cost by Category** table at the top, grouping spend by workload: **Chat** (student chat and quiz), **Voice (Realtime)**, **Voice (TTS)**, **Voice (STT)**, **RAG** (embedding calls), **Analytics** (Learning Radar admin queries).
- `meta_ai_sse.php` now logs Learning Radar queries with `interaction_type = 'meta'` and approximate prompt/completion tokens so they show up under the Analytics category. Prior to this release, Learning Radar spend blended invisibly into Chat.
- Existing per-model and per-student tables stay as-is; the new category table answers "how much of this bill is voice vs chat vs RAG" at a glance.

### Learning Radar (renamed from "AI Analysis Chat")

- User-facing label changed across 9 source files and all 46 lang files (37 localized translations, 8 kept the English brand verbatim for rare languages). Config keys, class names, DB columns, and HTML element IDs stay unchanged to avoid Moodle upgrade migrations.
- **6 metric chips** now sit at the top of the Analytics Dashboard: Tokens (30d), Top-cost course (30d), Active students (7d), Voice minutes (30d), Negative ratings (7d), Integrity flags (open). Each chip is clickable and submits a templated follow-up query to Learning Radar.
- **SOLA_NEXT follow-up suggestions.** After every Learning Radar response, the LLM emits a `[SOLA_NEXT]…[/SOLA_NEXT]` block with four short follow-up questions. The parser strips the tag and renders each suggestion as a clickable chip beneath the response; clicking a chip submits it. Same pattern SOLA uses for student chat, now for admin analytical drill-down.

### Testing

- PHP lint clean on every modified file (81 source + 46 lang).
- Deployed and verified on dev.sylr.org, dev405.sylr.org, dev500.sylr.org (Moodle 5.0), dev501.sylr.org (Moodle 5.1 with `public/` layout).

## Part 3: 20-minute Admin Walkthrough Script

Refer to `.drafts/v3.9.6-release-notes-and-walkthrough.md` Part 3 for the baseline walkthrough. v3.9.8 changes to the script:

- **Minute 15 (Testing Environment → Analytics demo).** When you open the Analytics Dashboard, now called out:
  - The 6 metric chips at the top of the page. Click "Top-cost course (30d)" to demo the chip → Learning Radar auto-submit path. Walk through the returned answer and the SOLA_NEXT chips that appear under it; click one to drill deeper.
  - The renamed **Learning Radar** card. Same functionality as the old AI Analysis Chat, with the new name.
- **Minute 16 (Safety).** Mention that the Token Cost page now separates voice and RAG spend from chat spend — useful for knowing where to trim if the bill surprises you.
