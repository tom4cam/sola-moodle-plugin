# SOLA v5.0.0 — May the Fourth Edition

> *May the Fourth be with you, and with your learners.*
> *These are the providers you are looking for. Stay on target.*

Major version bump that caps the structured-prompt arc shipped across v4.11.0 (consolidation), v4.12.0 (PromptSection + builder + budget), and the final tuning landed here. The cumulative effect on a typical course is a **~50% per-turn reduction** in assembled prompt size with no behavioural regressions. Coincidentally landing on Star Wars Day, hence the seasoning below; every claim in this document is real.

This release incorporates Tomi Molnár's UT rounds 2 through 6 audit feedback, Thelma's language-sync UT findings, a five-area provider-UX audit, and a study-plan audit triggered by a real reminder email Tom received from his own course. Twelve in-tag patches landed between the initial v5.0.0 cut and Catalyst's Monday production push, all under the same `v5.0.0` tag — these are the patches you are looking for.

## Tunes in v5.0.0 itself

- Default `prompt_budget_chars` raised from 8,000 → **12,000** (~3,000 tokens). The 8K default in v4.12.0 was tight for courses with rich FAQs and full personalization profiles; 12K gives the new `current_page_content` section room to land alongside personalization + multilingual + topic_focus. Code-level fallback updated to match.
- `multilingual` section priority raised from 40 → **70** so non-English learner courses do not lose this guidance under default budget pressure.

## Theme 2 — topic focus and conversation memory (Tomi UT round 2)

- New **`topic_focus`** section (LEARNER, priority 80). Replaces the legacy informational `## Current Page` injection with a directional `## Current focus` section that names the chapter the learner is on, tells the model to prioritise it, and adds the explicit redirect rule. Carved out so basic clarifying questions do not trigger over-eager redirects.
- New **`memory_handling`** section (BEHAVIOR, priority 60). Explicit rules for how the model should use conversation history: build on prior turns, don't promise persistence past the 50-turn cap, don't claim cross-session memory, recognise empty history as a clean reset.

## Theme 3 — language fixes (Thelma UT)

- `speech.js getLang()` now falls back to Moodle session language (`M.cfg.language`) before `navigator.language`. Fixes a learner with Moodle UI in English seeing SOLA respond in Spanish because `navigator.language` was `es-*` on their device.
- `get_multilingual_instructions()` strengthened: when an explicit lang is set, the directive overrides not just user input language but also any prior conversation-history language.

## Theme 4 — Clear button and page-grounding (Tomi UT round 2)

- `chat.js handleClear()` now calls `Repo.clearHistory(courseId)` server-side before the local UI wipe. Previously the Clear button only wiped the visible drawer; the conversation persisted in the DB and old turns kept being included in subsequent prompts. This was the root cause of the "still see Garfield in the chats" report.
- New **`current_page_content`** section (CONTEXT, priority 95) folded into the structured assembly with a strengthened directive ("Page-grounded answer required ... The page content takes precedence over your prior conversation turns and over any persona styling.") next to the content itself.
- **Persona carve-out** added to `get_security_instructions`: persona styling, coaching tone, and conversational character MUST NOT override factual answering of course-content questions.

## Theme 5 — prompt metrics, budget recommendation, Together AI rate card

- New `prompt_metrics_logger` writes one JSON line per chat turn to `moodledata/sola_prompt_metrics/YYYY-MM-DD.log` (rolling, 7-day retention; no PII, just per-section char counts). Default on; admin-disableable via `prompt_metrics_enabled`.
- New admin page `/local/ai_course_assistant/prompt_metrics.php` aggregates the last 7 days for per-category averages, surfaces a budget recommendation, and offers a one-click "Apply recommendation" button.
- New `prompt_budget_auto_tune` admin flag (default off): when on, a daily cron task auto-applies the recommendation. The recommendation always shows on the page; auto-apply is opt-in.
- LLM rate card extended with Together AI Llama 3.1 8B / 70B / 405B Instruct Turbo (Saylor's chosen production model is `meta-llama/Llama-3.1-8B-Instruct-Turbo` at $0.18/M flat).

## Theme 6 — final-prompt logging and page-grounding recency (Tomi UT round 3)

- `prompt_debug_enabled` log now writes the **full payload** the model receives this turn: assembled system prompt + per-section breakdown + conversation history array + the current user message + attachment metadata. The previous form dumped only the system prompt, which hid history-driven drift. *(You will be surprised what your droids have been seeing.)*
- New **`page_grounding_reminder`** SAFETY-tail section (priority 50). The page content + first directive still sit at CONTEXT priority 95, but chat-tuned LLMs over-weight the FIRST instruction. Re-emitting the directive at the very end of the prompt makes recency bias work in our favour, so a heavy admin systemprompt persona no longer overrides page-grounded answering.

## Theme 12 — study-plan polish (patch 12)

- **`hours_per_week` is now range-validated** in `update_study_plan`. Previously any float saved without complaint, so a learner committing to "100" or "168" hours per week (the latter a common STT failure when the model dictates in seconds) saved nonsense and then got reminders built around it. Now clamped to `[0.5, 60]` with a friendly chat-surface error.
- **Study-tip pool moved from a hardcoded English array to lang strings** (`studytip:pomodoro`, `studytip:active_recall`, etc.). Eight tips, all eight reachable for translators. Non-English reminder emails now reach learners in their language instead of leaking English copy.
- **Reminder email body richer when preferred days / time of day are set.** New `reminder:email_body_with_prefs` template renders "Your study plan: 8 hours per week, on Mon, Wed, Fri (evenings)." `reminder_manager` picks among three variants based on what the plan record contains.

## Theme 11 — study-plan reminder email no longer leaks "(not set)" (patch 11)

- Previously the body of the reminder email read "Your study plan suggests (not set) hours per week for this course" when the learner had never created a plan record (or had cleared one). The literal `(not set)` string was a hardcoded fallback in `reminder_manager` that leaked through `get_string` substitution into the learner's inbox. *(Don't underestimate the small things.)*
- Replaced with a separate `reminder:email_body_no_hours` template that omits the sentence entirely. PHP picks the right variant based on whether the plan exists with non-zero hours.

## Theme 10 — prompt debug log viewer (Tomi UT round 6, patch 10)

- New admin page at `/local/ai_course_assistant/prompt_debug_view.php` that renders the rolling per-turn debug log directly in the browser. Saves admins from SSHing into the server to read `moodledata/temp/sola_prompt_debug.log` by hand.
- Each entry renders as a collapsible card with timestamp, courseid, userid, provider, total / budget, per-section breakdown, full assembled system prompt, conversation history, current user message, and attachment metadata. Quick-scan badges on the card header flag whether the `current_page_content` and `topic_focus` sections actually landed in the prompt this turn — the most common "why didn't SOLA see my page?" diagnosis answered in two seconds.
- Triggered by Tomi's UT round 6: he reported page context not appearing in the prompt and asked for the exact full prompt the AI receives. The full payload is already captured (Theme 6, patch 6); admins just had no easy way to see it. *(These are the prompts you were looking for.)*

## Theme 9 — system-prompt authority over history (Tomi UT round 5, patch 9)

- New SAFETY-priority rule in `get_security_instructions()` declaring that the model's identity, persona, and tone are defined by the **current** system prompt — not by prior responses in conversation history. If history shows a persona that contradicts the current rules, treat it as stale drift and respond from the current rules.
- Triggered by Tomi's UT round 5: he reset a heavy "Garfield EN" custom systemprompt back to the default and the chat kept emitting Garfield because the conversation history still held six prior assistant turns in that voice. The Clear button is the operational fix (it wipes server-side history); this rule is the model-side fix so persona drift fades naturally instead of persisting until the 50-turn cap rolls it off. *(There is no persona but the one we choose. Search your feelings.)*

## Theme 8 — output cleanliness rule (Tomi UT round 4, patch 8)

- New SAFETY-priority rule in `get_security_instructions()` forbidding planning preambles ("Okay, let's respond to ..."), labelled wrappers ("Assistant Response:", "Final Answer:"), meta-commentary blocks ("Explanation of Choices:"), and any narration of the model's own reasoning. The rule explicitly overrides custom system prompts that ask for a structured "reasoning then answer" format.
- Triggered by an admin-customised "Garfield EN" persona template Tomi was testing with — the transcript showed every assistant turn emitting its full chain-of-thought scaffolding directly to the learner. The persona carve-out shipped in patch 6 only fires for course-content questions, so small talk like "good morning" was leaving the meta-commentary unchecked. This rule plugs that gap. *(These are not the meta-comments you are looking for.)*

## Theme 7 — multi-LLM provider UX (provider-area audit, patch 7)

- **Together AI is now a first-class chat provider**, not just a rate-card entry. New `together_provider` class extending `openai_compatible_provider`; default model `meta-llama/Llama-3.1-8B-Instruct-Turbo` (Saylor's production chat model); default base URL `https://api.together.xyz`. Wired into the chat dropdown, the LLM-comparison admin picker, the factory dispatch in `base_provider::instantiate`, and propagated to all 46 language files.
- **Chat / RAG / Voice settings now state their scope explicitly** so admins do not have to read code to figure out what affects what. Chat provider help text spells out that voice / TTS / STT live elsewhere and currently support only OpenAI or xAI. RAG embedding provider help text spells out that switching providers requires a re-index and that Together does not yet expose an embedding endpoint. Voice providers heading directs Saylor sites running Together / Anthropic / DeepSeek / Gemini / Mistral for chat to configure at least one voice row here. *(No more "I have a bad feeling about this" two weeks into a deployment.)*
- **Analytics provider-comparison panel renamed** from "AI Provider Comparison" to "Chat Provider Comparison" with help text noting voice and RAG comparisons are on the v5.1 roadmap.

## The structural arc this release closes

- v4.11.0 — consolidated 5 scattered marker explanations into one `## Output markers` block; merged brevity + AI literacy + wellbeing + struggle detection into one `## House style` block; collapsed Socratic mode to a 1-line directive; added `prompt_debug_enabled`.
- v4.12.0 — replaced the linear concat with a list of `PromptSection` objects assembled by category (identity / context / learner / behavior / markers / safety) within an admin-configurable budget.
- v5.0.0 — tuned the defaults, layered in Tomi UT rounds 2 and 3 + Thelma UT, added the metrics surface and Together AI rate card.
