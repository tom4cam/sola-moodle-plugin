# SOLA v5.0.0 — Release Notes

Major version bump that caps the structured-prompt arc shipped across v4.11.0 (consolidation), v4.12.0 (PromptSection + builder + budget), and the final tuning landed here. The cumulative effect on a typical course is a **~50% per-turn reduction** in assembled prompt size with no behavioural regressions.

## Tunes in v5.0.0 itself

- Default `prompt_budget_chars` raised from 8,000 → **10,000** (~2,500 tokens). The 8K default in v4.12.0 was tight for courses with rich FAQs and full personalization profiles; 10K gives ~25% more headroom while still cutting prod prompt sizes by ~40% vs the v4.10 baseline. Code-level fallback updated to match.
- `multilingual` section priority raised from 40 → **70** so non-English learner courses do not lose this guidance under default budget pressure. The section is small (~600 chars) and load-bearing for the ESL population SOLA serves heavily.

## The structural arc this release closes

- v4.11.0 — consolidated 5 scattered marker explanations into one `## Output markers` block; merged brevity + AI literacy + wellbeing + struggle detection into one `## House style` block; collapsed Socratic mode to a 1-line directive (verbose form available via admin flag); added `prompt_debug_enabled` for empirical measurement.
- v4.12.0 — replaced the linear concat in `build_system_prompt` with a list of `PromptSection` objects assembled by category (identity / context / learner / behavior / markers / safety) within an admin-configurable budget. Per-section priorities + truncation floors + safety-exemption rules. Sections drop or truncate deterministically under budget pressure.
- v5.0.0 — tuned the defaults based on dev measurements.

## Empirical impact (TEST101 dev course)

| Release | Assembled prompt | Tokens |
|---|---|---|
| v4.10 baseline | ~17,126 chars | ~4,281 |
| v4.11.0 lean | 16,336 chars | 4,084 |
| v4.12.0 default (8K budget) | 7,908 chars | 1,977 |
| **v5.0.0 default (10K budget)** | **9,244 chars** | **2,311** |

v5.0.0's slightly larger size vs v4.12 is intentional — it preserves multilingual + practice_scoring sections that v4.12 was dropping. Production courses without TEST101's unusually large 4.4K-char FAQ will land closer to 7K chars at the 10K budget.

## Testing

- PHP lint clean on every modified file.
- Validator suite: 30/30 pass.
- Empirical prompt size measurement on TEST101: confirmed multilingual + practice_scoring now preserved at default 10K budget.

## Backwards compatibility

Zero breaking changes. Existing custom system-prompt overrides via the `systemprompt` admin setting still work. Talking-avatar provider integrations (v4.9 + v4.10) and avatar cost tracking (v4.10 schema) unchanged.
