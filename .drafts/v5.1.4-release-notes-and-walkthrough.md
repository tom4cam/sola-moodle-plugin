# SOLA v5.1.4

Two small UX/quality fixes.

## What's fixed

**SOLA_NEXT follow-up chips were silently falling back to JS-hardcoded defaults.** The dynamic suggestion chips at the bottom of every assistant response (`Tell me more / Give me an example / Quiz me on this` were the giveaway) were rendering the fallback strings on most turns because smaller chat-tuned models (Llama 3.1 8B, Mistral Small, Gemma) were under-following the one-line bullet inside the `## Output markers` block of the system prompt.

Fix: promoted the SOLA_NEXT requirement to a SAFETY-priority output rule in `get_security_instructions`. The rule:
- Always lands (SAFETY-priority sections are exempt from budget-pressure dropping).
- Sits at the recency-bias tail of the prompt, where chat-tuned models pay the most attention.
- States the exact marker shape unambiguously: `[SOLA_NEXT]suggestion 1||suggestion 2||suggestion 3||suggestion 4[/SOLA_NEXT]`.
- The MARKERS-section bullet stays in place for redundancy, but this is now the enforcement copy.

## What's improved

**Reading-level setting in the learner settings panel** (now called "Explanation style"). The previous label "Reading level" with grade-level option text confused learners. Renamed and clarified:

- New label: **Explanation style**
- New help text: "How simply should SOLA explain things? The course content stays the same — only the way SOLA explains it changes."
- New option labels:
  - "Match the course (default)" (was "Default (course level)")
  - "Easier — short sentences, simple words (about middle-school level)" (was "Plain language (about 6th grade)")
  - "Plain — clear language, define any jargon (about high-school level)" (was "Plain language (about 12th grade)")

Underlying behaviour is unchanged — the same `simple` / `standard` keys flow into the SSE request and the system prompt picks them up exactly as before.

## Backwards compatibility

Drop-in upgrade from v5.1.3. No setting changes, no schema changes. Two affected files: `classes/context_builder.php` and `amd/src/ui.js` (and its rebuilt `amd/build/ui.min.js`).

Plugin version `2026050504`, release `5.1.4`.
