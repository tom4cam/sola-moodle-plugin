# SOLA v5.1.5

Two small UX cleanups in the learner Settings panel.

## What's fixed

**Settings tour referenced a non-existent section.** The first-open tour card said:

> The "AI helper" section near the bottom is optional — leave it on the default if you are not sure.

There has not been an "AI helper" section in the panel for several major versions; the copy was stale from a pre-v4 rename that was never updated. Learners reasonably went looking for a section that did not exist.

Rewrote the tour copy to be generic across the actual sections that render (Language, Coaching Style, Avatar, Accessibility, SOLA Voice, etc.) so future section renames do not falsify the tour again:

> Pick a language, choose an avatar, set how SOLA explains things, and tune voice and accessibility options. Changes here only affect how SOLA appears for you. Every setting has a sensible default — feel free to tweak only the ones that help you, and revisit later.

## What's removed

**"Saved responses" section in the Settings panel.** Bookmarked responses already surface in the Notes tab where learners can read, edit, and organise them with the same actions, so a second copy under Settings was confusing. The bookmark store APIs (`getBookmarks`, `removeBookmarkByText`) and the star toggle on each message are unchanged — the Notes tab continues to read the same data.

## Backwards compatibility

Drop-in upgrade from v5.1.4. No setting changes, no schema changes. Single file diff in `amd/src/ui.js` (and its rebuilt `amd/build/ui.min.js`).

Plugin version `2026050505`, release `5.1.5`.
