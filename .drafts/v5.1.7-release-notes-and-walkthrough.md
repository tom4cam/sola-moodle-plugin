# SOLA v5.1.7

UX hotfix.

## What's fixed

Three per-course pages returned a hard Moodle 404 when an admin opened the URL without a `courseid` query parameter:

- `/local/ai_course_assistant/instructor_dashboard.php`
- `/local/ai_course_assistant/objectives_admin.php`
- `/local/ai_course_assistant/sandbox.php`

The pages were calling `required_param('courseid', PARAM_INT)`, which throws when the param is missing and Moodle renders a 404. Confusing for an admin who just wanted to navigate to the page from the admin tree.

Fix: replaced with `optional_param('courseid', 0)` plus a friendly course-picker landing. New helper `page_helpers::render_course_picker_landing` lists every course the current user has the relevant capability in (site admins see every visible course), each linking back to the same page with `?courseid=N` filled in.

Direct links with a courseid parameter are unchanged.

## Backwards compatibility

Drop-in upgrade from v5.1.6. No setting changes, no schema changes. Three new lang strings (`coursepicker:title`, `coursepicker:intro`, `coursepicker:nocourses`) propagated to all 46 lang files. New file `classes/page_helpers.php`.

Plugin version `2026050507`, release `5.1.7`.
